1. GETTING STARTED WITH THE APPLICATION. INTERFACE OF THE MAIN WINDOW

When you start the CryptoARM GOST application, the main application window appears, as shown in the figure below. On the main window there are buttons for navigating to the main application wizards: the wizard for creating / verifying an electronic signature, the Encryption / Encryption Wizard, the Certificate Management Wizard for the connected storage.

In the title bar of the application there is a button for calling the side menu (the menu is shown in the figure to the right) and a button switching the interface language. Using the application menu, you can go to the main wizards and optionally get access to the page with information about the program, the license installation wizard and quick help.



The main application window has a fixed size without the possibility of changing it. The application saves all global settings that were made by the user. Saving occurs when the application is closed. In progress with the application saved, the user's actions are also saved, executed in the wizards (selection of certificates, selection of files, etc.). When you go to another wizard (application page) and return to the application, you do not have to perform the same operations again.



2. MASTER OF LICENSE INSTALLATION

For the correct operation of the application, you must install a license for this software product. If, at the application startup, there are pop-up messages about the absence of a license expired in the license period, impossibility to read the license file, this indicates that in this application will be incorrectly executed operation of signing and decrypting files. The presence of a valid license for the application should be given special attention.

To install the license and get information about it in the application there is a special wizard (shown in the figure below), which allows you to install a license from a file or clipboard. In either case the license is added to the license.lic file and placed in the user's directory.



Important! After the license has been successfully installed, the program must be restarted. If you type the following messages when you enter a license for a software product: a non-correct license key, the license is not suitable for this program, the license has expired, then you should contact technical support company-developer (info@trusted.ru).



3. MASTER OF CREATION / ELECTRONIC SIGNATURE CHECKS

In the application Crypto ARM GOST there is a wizard in which the operations of creating an electronic signatures for one or multiple files. The same wizard implements the creation operations file signing and verification of signed files. The wizard page is shown in the figure below.

The wizard page is divided into three areas. On the left is the selection area of the subscriber certificate. The certificate is selected from the dialog with the display of certificates from the repositories.

Важно! In the dialog for selecting a subscriber certificate, only certificates from Personal certificates that have a binding to the private key. Other certificates are not displayed and are not available for use in this wizard.



To verify the signature, it is enough to select the files to be scanned - files with the extension .sig, which contain the electronic signature and click on the Check button. No additional manipulations in verifying the signature do not need to produce.

Important! If the source file is not found when checking the signature file separated from the signature automatically, his choice will be offered. The result of checking signatures is displayed as a message.

To view the files before creating a signature, you can use the context menu (see the figure below). The signature is checked for the selected files. The figure shows the result of the signature verification one of the files. Additionally, you can remove the signature from the signed files. In this case, the original file is saved, and the signature is deleted.




4. MASTER OF ENCRYPTION / DISCRIPTION OF FILES

In the application Crypto ARM GOST there is a wizard in which the operations of encryption and decryption of files are performed. To perform the encryption operation, the recipients of the encrypted files are indicated in the form of a list of certificates. Those users of the application who have a private key associated with one of the recipient certificates will be able to decrypt the files.

The Encryption / Decryption Wizard is shown in the figure below. The wizard page is divided into three areas: On the left is the selection area of the recipient certificates, the settings for the operations are located below and the right is located the area where the source files are selected and the result displayed.

Important! When selecting the mode for deleting files after encryption, pay attention to providing access to encrypted files. To do this, you can add a personal certificate to the list of recipients, linked to a private key.

The figure at the right shows the dialog for selecting recipient certificates. In the list of available certificates all certificates from the connected storage are displayed. The certificates are grouped by tabs. In the dialog, an operation is available search for certificates.



In the wizard, you can modify the list of recipients and files by adding them. Encryption operation is performed only over a set of selected files. To view files before encryption, you can use context menu (see the figure below).

Decrypt files by clicking the Decrypt button. To perform the operation successfully in one of the connected storages there must be a private key with a binding to the certificate of the recipient. Otherwise an error will be generated.

If the files are successfully decrypted, the result is displayed in the same wizard and the files are saved either side by side with encrypted, or in the directory specified in the settings.




5. MASTER OF CERTIFICATES MANAGEMENT AND KEYS

To manage certificates and keys, a separate wizard has been added to the application. On this page wizards, tabs are displayed with the certificates grouped in them. When you first start the application in the tab Personal certificates must have a test certificate and a key attached to it.

In order to import the new certificate into the repository, you can use the context menu - select the Import Certificate operation. In the dialog that appears, select certificate (supported by the BASE64 and DER encodings) and the certificate is placed in the repository. The imported in the certificate store there is no binding to the private key. In order to import a closed key and link it to the selected certificate, you need to call the context menu and select Import key. In the dialog box that appears, select the private key file and it will be bound to the certificate.

Important!The current version of the program supports the import of a private key from the format container PKCS # 8 without password. Import is performed in the system's own application storage. Import certificates from storage of supported providers is carried out at the level of using additional utilities. Most often the situations that occur are explained in the User's Guide.
